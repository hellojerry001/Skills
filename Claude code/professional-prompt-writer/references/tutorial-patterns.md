# Tutorial-Style Prompt Patterns

教程类内容用于教育用户正确理解和使用某个概念、工具或方法。特点是结构化、洞察深刻、小白友好。

## 核心结构

### 基础框架
```
标题：[吸睛的标题 + 核心价值]
↓
引言：[揭示常见问题/痛点]
↓
主体：[误区列举 或 步骤指导]
↓
金句总结：[升华认知的核心洞察]
```

## 模式 1：误区纠正型

适用于：纠正用户对某个概念/工具的错误理解

### 结构模板
```
# [主题]必看！[N]个误区教你真正掌握[核心概念]

[引言段落]
- 揭示大部分人的错误做法
- 点明这种错误的本质
- 预告将要讲解的内容

今天通过[N]个误区让你彻底掌握[主题]

误区一：[误区标题]
[简短解释为什么这是误区]

误区二：[误区标题]
[简短解释为什么这是误区]

误区三：[误区标题]
[简短解释为什么这是误区]

[金句总结段落]
- 用一句话总结核心认知
- 点明正确理解的本质
- 给出行动建议或判断标准
```

### 写作要点
1. **引言要有张力**：用"大部分人认为...但这其实..."制造认知反差
2. **误区要具体**：不是泛泛而谈，而是指出具体的错误行为
3. **金句要升华**：从具体误区升华到本质认知
4. **语气要亲切**：用"你"而不是"我们"，用"其实"、"但是"等口语化词汇

## 模式 2：步骤指导型

适用于：教授完成某个任务的正确方法

### 结构模板
```
# [主题]必看！[N]步教你如何[达成目标]

[引言段落]
我发现很多人[做某事]
[点明错误做法]

这样[做]到的根本不是[预期结果]
而是[实际得到的结果]

如果你只[错误做法的核心]，
那你永远只是在[负面结果]

[N]个步骤教你如何[正确方法]

第一步：[步骤标题]
[具体操作指导和原理]

第二步：[步骤标题]
[具体操作指导和原理]

第三步：[步骤标题]
[具体操作指导和原理]

[金句总结段落]
所以，[核心概念]从来不是[表面理解]，
而是[深层本质]。

当你能[正确做法]，而不是[错误做法]，
你才真正[达成目标]。
```

### 写作要点
1. **步骤要清晰**：每个步骤有明确的动作指向
2. **对比要鲜明**：强调"不是...而是..."的对比
3. **原理要点透**：不只说怎么做，还要说为什么
4. **金句要呼应**：总结段落呼应引言的痛点

## 语言风格指南

### 开头常用句式
- "我发现很多人..."
- "大部分人认为..."
- "你是不是也经常..."

### 转折常用词汇
- "但这，其实是..."
- "而是..."
- "真正的...是..."

### 金句常用结构
- "[概念]从来不是[X]，而是[Y]"
- "当你能[A]，而不是[B]，你才真正[目标]"
- "真正[形容词]的结果，来自[X]，而不是[Y]"

## 质量检查清单

生成教程内容后，检查：
- [ ] 引言是否揭示了真实痛点？
- [ ] 误区/步骤是否具体可感知？
- [ ] 每个部分是否有独特价值（不重复）？
- [ ] 金句总结是否升华了认知层次？
- [ ] 语言是否口语化、亲切？
- [ ] 结构是否清晰、易扫读？
