## 教程类案例

### 案例 1：AI 参考图使用教程

```
小白必看！三个误区教你真正掌握ai参考图

我发现很多人用参考图第一步就错了

大部分人认为

我给模型一张图，它就应该生成这个结果

但这，其实是人的理解

并不是 AI 的工作方式

在 AI 眼里参考图从来不是成品 而是一种特征

今天通过三个误区让你彻底掌握ai参考图的用法


误区一：把参考图当成成品目标。

误区二：一张图想教太多内容

误区三：参考图不是用来弥补文本问题的


最后记住一句话： 参考图的本质不是增加灵感，而是减少自由度


真正高质量的结果

来自对生成空间的精准约束， 而不是不断叠加参考信息。

如果你发现参考图越加越乱

那问题几乎一定不在模型

而在你使用它的方式
```

#### 分析
- **类型**：误区纠正型
- **结构**：引言（揭示错误认知）→ 三个误区 → 金句总结
- **亮点**：
  - 开头直击痛点："第一步就错了"
  - 认知反差："人的理解 vs AI的工作方式"
  - 金句升华："减少自由度"而非"增加灵感"
  - 最后给出判断标准：如果越加越乱，问题在使用方式

### 案例 2：AI 风格提取教程

```
小白必看！三步教你如何正确提取AI风格内容

我发现很多人提取画面风格

就是把图丢给 AI让它反推提示词

这样提取到的根本不是风格

而是那一次生成的结果

如果你只盯着画面本身，

那你永远只是在抄

三个步骤教你如何正确提取画面风格


第一步：不要看画面有什么，而是看没什么

第二步：让ai看共性而不是看独立。

第三步，复用画面结构搭建体系


所以，风格从来不是提示词，

而是一套被反复执行的选择逻辑。

当你能复用这套逻辑，而不是复刻某一张画面，你才真正拥有了这个风格。
```

#### 分析
- **类型**：步骤指导型
- **结构**：引言（点明错误做法）→ 三个步骤 → 金句总结
- **亮点**：
  - 痛点明确："永远只是在抄"
  - 步骤有洞察力：看"没什么"、看"共性"、搭建"体系"
  - 金句点透本质："选择逻辑"而非"提示词"
  - 对比鲜明："复用逻辑 vs 复刻画面"

## 专业指令类案例

### 案例 3：UI 设计系统分析

```
Deeply analyze the design of the attached screenshot to create a design system layout of every UI component needed in a design system for a SaaS product service.

Pay close attention to:
- spacing
- fonts
- colors
- design style
- design principles
```

#### 分析
- **类型**：设计分析型
- **结构**：任务描述 + 关键要素列表
- **亮点**：
  - 动词精准："Deeply analyze"（深度分析）
  - 目标明确："create a design system layout"
  - 上下文清晰："for a SaaS product service"
  - 要素完整：覆盖 spacing、fonts、colors、style、principles
  - 简洁有力：核心信息不超过 3 句话

## 使用这些案例的方法

### 1. 识别模式
每个案例代表一种特定的 Prompt 模式，使用时先识别用户需求属于哪种类型。

### 2. 提取结构
不要照搬文字，而是提取其背后的结构：
- 案例1：引言 + 误区列表 + 金句
- 案例2：引言 + 步骤列表 + 金句
- 案例3：任务 + 要素列表

### 3. 复用技巧
- **认知反差**："大部分人认为...但这其实是..."
- **对比升华**："不是 X，而是 Y"
- **判断标准**："如果出现 X，那问题在 Y"
- **本质揭示**："XX 从来不是 A，而是 B"

### 4. 调整语气
根据目标受众调整：
- 小白用户：更多口语化、类比、"你是不是也..."
- 专业用户：更简洁、直接、专业术语
